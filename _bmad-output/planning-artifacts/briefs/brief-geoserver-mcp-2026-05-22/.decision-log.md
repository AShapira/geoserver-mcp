# Decision Log: GeoServer MCP Product Brief

Created: 2026-05-22

## Decisions

- 2026-05-22: Create a greenfield product brief for an MCP server that exposes GeoServer capabilities across multiple GeoServer instances, including OGC services, REST API operations, and extension-aware capabilities.
- 2026-05-25: Primary audience is open-source GIS and IT professionals.
- 2026-05-25: First use case is AI-agent inspection and management of multiple GeoServer instances through MCP.
- 2026-05-25: Initial implementation should start read-only, with safe write/admin actions deferred to later phases.
- 2026-05-25: The brief should be strong enough to drive the downstream BMAD PRD and architecture work.
- 2026-05-25: Use Fast path for the product brief: one consolidated question batch, then draft with explicit assumptions where needed.
- 2026-05-25: Primary user focus is admin-first, while still serving GIS analysts, DevOps/SRE, and developers.
- 2026-05-25: v1 MVP focuses on inventory and diagnostics; usage statistics are deferred to future steps.
- 2026-05-25: v1 baseline access is HTTP/HTTPS with administrator or equivalent permissions. Local data-directory access may be used when available but must not be assumed.
- 2026-05-25: Configuration should support both config files and environment variables.
- 2026-05-25: v1 authentication starts with basic auth.
- 2026-05-25: Future management actions should prioritize CRUD, especially catalog resources.
- 2026-05-25: Product positioning is both a GeoServer operations assistant for AI agents and a diagnostic/governance tool.

## Open Questions

- Exact list of diagnostic checks for v1.
- Whether TypeScript, Python, or another MCP SDK is preferred for implementation.
- Which GeoServer versions should define the initial compatibility target.
- Whether optional data-directory inspection should ship in v1 or be designed in v1 and implemented later.
